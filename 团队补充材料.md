## 一 · 开题报告补充材料

开题报告正文（Part 1 / Part 2）见报名表。这里补充评委最可能追问的问题与我们的应答。

**答辩备题**

1. **「主动」和传统告警的区别？** 传统是越限才报；本方案没报警时也在巡检 SPC 趋势——参数还在规格内、刚出现连续单边或 Cpk 下滑，就前瞻立案。是预测，不是事后。
2. **「确定性」怎么落地、不是口号？** 一道确定性校验门做三态判定：证据可回溯且不冲突 → 自动执行低风险处置并复测；不足或矛盾 → 转人工；本体外实体 → 拦截。再用 SPC / Cpk 把过程能力受控作为质量学意义上的「确定」。
3. **会不会乱停线？** 告警先分级、聚合降误报；停线、改参数等高风险动作一律人工确认并留痕。宁可漏报转人工，不乱下发。
4. **数据从哪来？** 复用产线既有质检数据（视觉 / CT / 参数）；落地第 0 步先做数据可得性盘点与量具校准（MSA），把「数据可信」作为「结论可信」的前置门。
5. **为什么用 Graph RAG？** 一个报警的根因常横跨多道工序，向量检索按相似度召回单点、容易断链；图检索顺着因果链，才能把跨工序的根因串全。

**建议 KPI**（均为厂内可自测、与质量强相关）：漏检率 DPPM · 误报率（虚警/千次）· 根因命中率 · 处置闭环时延 · 证据链可追溯率 · 过程能力 Cpk 回升。

## 四 · 研究笔记（关键决策）

- **为什么用半导体数据证明汽车命题** — 海选拿不到赛力斯真数据；我们不编假汽车数据，而用真实工业数据证明引擎跑得通。引擎与领域解耦——迁移只需换数据接口 + 工艺本体。
- **为什么要「确定性 gate」** — 大模型判断是概率性的，工业质量场景不能照单全收。把它收敛成三态确定结果，是本方案与普通问答智能体的根本差异。
- **一个被修掉的漏洞** — 三态判定曾有「证据齐全但错」的缺口（知识库过时 / 伪根因自洽会被无兜底执行）；已补上「自动执行后复测确认，未恢复则自动转人审」。
- **主动 vs 被动** — 用 SPC 趋势规则（连续单边 / Cpk 下滑）在越规格之前介入，即前瞻立案。
